## Provider Keys

| Variable | Required | Description |
|---|---|---|
| `ANTHROPIC_API_KEY` | Yes (for Anthropic) | API key. Aliases: `ANTHROPIC_KEY`, `ANTHROPIC_TOKEN`, `CLAUDE_API_KEY` |
| `OPENAI_API_KEY` | Yes (for OpenAI) | API key. Aliases: `OPENAI_KEY`, `OPENAI_TOKEN` |
| `GOOGLE_API_KEY` | Yes (for Gemini) | API key for Google Gemini |
| `GROQ_API_KEY` | Yes (for Groq) | API key for Groq (OpenAI-compatible) |
| `MOONSHOT_API_KEY` | Yes (for Moonshot) | API key for Moonshot/Kimi (OpenAI-compatible) |
| `OLLAMA_API_KEY` | No | Optional API key for Ollama |

Other OpenAI-compatible providers follow the same pattern.

## Server Configuration

| Variable | Default | Description |
|---|---|---|
| `ANIMAOS_RS_HOST` | `127.0.0.1` | Bind host |
| `ANIMAOS_RS_PORT` | `8080` | Bind port |
| `ANIMAOS_RS_MAX_REQUEST_BYTES` | `65536` | Request body size limit in bytes |
| `ANIMAOS_RS_REQUEST_TIMEOUT_SECS` | `30` | Per-request timeout |
| `ANIMAOS_RS_MAX_CONCURRENT_RUNS` | `8` | Max concurrent `/run` requests before `503` |
| `ANIMAOS_RS_MAX_BACKGROUND_PROCESSES` | `8` | Max concurrent `bg_start` processes |

## Persistence

| Variable | Default | Description |
|---|---|---|
| `ANIMAOS_RS_PERSISTENCE_MODE` | `memory` | `memory` or `postgres` |
| `DATABASE_URL` | — | Postgres connection string (required for `postgres` mode) |
| `ANIMAOS_RS_CONTROL_PLANE_FILE` | — | JSON snapshot file for registered agents, swarms, and latest runtime states |
| `ANIMAOS_RS_MEMORY_FILE` | — | JSON snapshot file for runtime memories |
| `ANIMAOS_RS_MEMORY_SQLITE_FILE` | — | SQLite snapshot file for memories + embeddings |

<Aside type="note">
  Set only one of `ANIMAOS_RS_MEMORY_FILE` or `ANIMAOS_RS_MEMORY_SQLITE_FILE`. If both are set, SQLite takes precedence.
</Aside>

## Memory Embeddings

| Variable | Default | Description |
|---|---|---|
| `ANIMAOS_RS_MEMORY_EMBEDDINGS` | `local` | `local`, `fastembed`, `ollama`, `openai`, `openai-compatible`, `disabled` |
| `ANIMAOS_RS_MEMORY_EMBEDDING_MODEL` | — | Model name (provider-specific) |
| `ANIMAOS_RS_MEMORY_EMBEDDING_DIMENSIONS` | — | Expected vector size |
| `ANIMAOS_RS_MEMORY_EMBEDDINGS_BASE_URL` | — | OpenAI-compatible embedding base URL |
| `ANIMAOS_RS_MEMORY_EMBEDDINGS_API_KEY` | — | API key for provider-backed embeddings |
| `ANIMAOS_RS_MEMORY_EMBEDDINGS_TIMEOUT_MS` | `15000` | HTTP timeout for embedding calls |
| `ANIMAOS_RS_MEMORY_EMBEDDINGS_CACHE_DIR` | — | Cache directory for `fastembed` models |
| `ANIMAOS_RS_MEMORY_EMBEDDINGS_SHOW_DOWNLOAD_PROGRESS` | `true` | Show model download progress |
| `ANIMAOS_RS_MEMORY_EMBEDDINGS_SQLITE_FILE` | — | Separate SQLite file for embedding vectors |

## Text Analysis

| Variable | Default | Description |
|---|---|---|
| `ANIMAOS_RS_MEMORY_TEXT_ANALYZER` | `multilingual` | BM25 text analyzer profile |

Profiles:
- `multilingual` / `unicode` — Language-neutral Unicode tokenization with CJK character/bigram terms

## Example Configurations

### Minimal (in-memory, local embeddings)

```bash
OPENAI_API_KEY=sk-... cargo run -p anima-daemon
```

### With SQLite persistence

```bash
ANIMAOS_RS_MEMORY_SQLITE_FILE=./data/runtime-memories.sqlite \
  OPENAI_API_KEY=sk-... \
  cargo run -p anima-daemon
```

### With control-plane restart recovery

```bash
ANIMAOS_RS_CONTROL_PLANE_FILE=./data/control-plane.json \
  ANIMAOS_RS_MEMORY_SQLITE_FILE=./data/runtime-memories.sqlite \
  OPENAI_API_KEY=sk-... \
  cargo run -p anima-daemon
```

### With Postgres + fastembed

```bash
ANIMAOS_RS_PERSISTENCE_MODE=postgres \
  DATABASE_URL=postgres://user:pass@localhost/anima \
  ANIMAOS_RS_MEMORY_SQLITE_FILE=./data/runtime-memories.sqlite \
  ANIMAOS_RS_MEMORY_EMBEDDINGS=fastembed \
  ANIMAOS_RS_MEMORY_EMBEDDING_MODEL=intfloat/multilingual-e5-small \
  ANTHROPIC_API_KEY=sk-ant-... \
  cargo run -p anima-daemon
```

### With Ollama embeddings

```bash
ANIMAOS_RS_MEMORY_SQLITE_FILE=./data/runtime-memories.sqlite \
  ANIMAOS_RS_MEMORY_EMBEDDINGS=ollama \
  ANIMAOS_RS_MEMORY_EMBEDDING_MODEL=nomic-embed-text \
  OPENAI_API_KEY=sk-... \
  cargo run -p anima-daemon
```
