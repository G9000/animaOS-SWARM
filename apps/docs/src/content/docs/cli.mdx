The animaOS CLI (`animaos`) is the operator entrypoint for animaOS. It handles two different jobs:

- local workspace generation through `create`
- daemon-backed execution through `launch`, `run`, `chat`, and `agents`

`launch` is the primary day-to-day workflow. It turns an agency workspace into a live daemon-backed session and can render the Ink TUI. `run` is the quick plain-text shortcut for one-off work. `chat` gives you a single interactive agent. `agents` inspects what the daemon is currently holding.

## Mental Model

Use the command that matches the job:

- `animaos create` builds a local agent or agency workspace. It does not require the daemon.
- `animaos launch` runs an existing `anima.yaml` workspace against the daemon. This is the main TUI flow.
- `animaos run` is the fastest way to send one task to a temporary agent or swarm.
- `animaos chat` opens an interactive single-agent REPL backed by the daemon.
- `animaos agents` lets you inspect daemon-backed agents already registered in the host.

## Fast Path

<Steps>

1. Start the daemon.

   ```bash
   bun run daemon
   ```

2. Generate an agency workspace.

   ```bash
   animaos create agency content-team \
     -d "Research topics and write high-quality articles" \
     -s 5 \
     --seed
   ```

3. Launch that workspace in the TUI.

   ```bash
   cd content-team
   animaos launch
   ```

</Steps>

## Command Guide

### `animaos create`

Create a single agent workspace or a full agency directory. This is a local filesystem workflow, not a daemon workflow.

```bash
# Interactive mode: choose agent or agency
animaos create

# Single-agent workspace
animaos create agent medicine-helper \
  -d "Research medical literature and summarize risks clearly"

# Full agency workspace
animaos create agency content-team \
  -d "Research topics and write high-quality articles" \
  -s 5 \
  --seed

# Legacy shorthand still creates an agency
animaos create content-team --yes
```

`create` uses configured remote provider credentials when they are available and falls back to a deterministic local scaffolder when no provider key is present.

| Flag | Default | Description |
|------|---------|-------------|
| `--kind` | — | `agent` or `agency`; equivalent to `animaos create agent ...` |
| `-p, --provider` | `auto` | Generation provider. Supports `deterministic`, `openai`, `anthropic`, `google`/`gemini`, `ollama`, `groq`, `xai`/`grok`, `openrouter`, `mistral`, `together`, `deepseek`, `fireworks`, `perplexity`, `moonshot`/`kimi` |
| `-m, --model` | `gpt-4o-mini` | Model for generation |
| `--models` | — | Comma-separated model pool for generated agents |
| `-d, --description` | — | What the agent or agency is supposed to do |
| `-s, --size` | `4` | Agency team size (2-10) |
| `--system` | — | System prompt for single-agent mode |
| `--seed` | — | Generate starter memories for each agent |
| `-y, --yes` | — | Accept the generated team without confirmation |

The generated directory is designed for human editing:

```text
content-team/
├── anima.yaml        # Team config used by launch
├── org-chart.mmd     # Mermaid org chart
├── README.md         # Mission, values, and roster
└── agents/
    ├── orchestrator/
    │   ├── profile.md
    │   ├── assets/
    │   └── memory/
    │       └── seed.json
    └── worker-1/
        ├── profile.md
        ├── assets/
        └── memory/
```

### `animaos launch`

Launch an agency workspace from its `anima.yaml`. This is the primary operator workflow and the main entrypoint for the TUI.

```bash
cd content-team

# TUI one-shot task
animaos launch "Write an article about Rust macros"

# TUI interactive session
animaos launch

# Plain-text one-shot task
animaos launch --no-tui "Write an article about Rust macros"

# Plain-text interactive session
animaos launch --no-tui
```

During launch, the CLI can read `agents/<slug>/memory/seed.json` files and post those memories into the daemon before work begins.

| Flag | Default | Description |
|------|---------|-------------|
| `--api-key` | — | API key override for the daemon-backed run |
| `--no-tui` | — | Disable the Ink TUI and use plain text instead |

Plain-text interactive launch supports:

| Command | Description |
|---------|-------------|
| `exit` | Quit the session |
| `/help` | Show the available commands |
| `/health` | Recheck daemon connectivity |

### `animaos run`

Run a temporary single agent or a temporary swarm for one task. This is the fastest CLI path when you do not want to create a local agency workspace first.

```bash
# Single agent
animaos run "Summarize the Rust book" -m gpt-4o-mini

# Swarm with supervisor strategy
animaos run "Build a landing page" -s supervisor -m gpt-4o
```

<Aside type="caution">
  `run` is plain-text today. The command keeps a `--no-tui` flag for interface consistency, but daemon-backed `run` currently falls back to plain text instead of rendering the TUI.
</Aside>

| Flag | Default | Description |
|------|---------|-------------|
| `-m, --model` | `gpt-4o-mini` | Model to use |
| `-p, --provider` | `openai` | Provider family for the daemon-backed run |
| `-n, --name` | `task-agent` | Agent name for single-agent mode |
| `-s, --strategy` | — | Swarm strategy: `supervisor`, `dynamic`, or `round-robin` |
| `--api-key` | — | API key override |
| `--no-tui` | — | Force plain-text output |

### `animaos chat`

Start a daemon-backed interactive REPL with one agent.

```bash
animaos chat -m claude-3-5-sonnet -p anthropic
```

Type messages at the `you >` prompt. Type `exit` to quit.

### `animaos agents`

Inspect daemon-backed agents that are currently registered in the host.

```bash
# List all agents
animaos agents list

# Show details for a specific agent by name or id
animaos agents show researcher
```

This is useful when you want to check what the daemon currently holds after runs, launches, or host restarts.

### `animaos mod`

Enable, disable, or list mods.

```bash
animaos mod enable my-mod
animaos mod disable my-mod
animaos mod list
```

## Provider Support

The CLI accepts the same main provider family across `create`, `launch`, `run`, and `chat`.

| Provider | Accepted names | Key env vars | Base URL env var | Notes |
|----------|----------------|--------------|------------------|-------|
| OpenAI | `openai` | `OPENAI_API_KEY`, `OPENAI_KEY`, `OPENAI_TOKEN` | `OPENAI_BASE_URL` | Native OpenAI chat completions |
| Anthropic | `anthropic` | `ANTHROPIC_API_KEY`, `ANTHROPIC_KEY`, `ANTHROPIC_TOKEN`, `CLAUDE_API_KEY` | `ANTHROPIC_BASE_URL` | Native Anthropic Messages API |
| Google Gemini | `google`, `gemini` | `GOOGLE_API_KEY`, `GOOGLE_KEY`, `GOOGLE_AI_KEY`, `GEMINI_API_KEY`, `GOOGLE_GENERATIVE_AI_API_KEY` | `GOOGLE_BASE_URL` | Native Google `generateContent` API |
| Ollama | `ollama` | `OLLAMA_API_KEY` | `OLLAMA_BASE_URL` | Local inference via OpenAI-compatible HTTP shape |
| Groq | `groq` | `GROQ_API_KEY`, `GROQ_KEY`, `GROQ_TOKEN` | `GROQ_BASE_URL` | OpenAI-compatible |
| xAI | `xai`, `grok` | `XAI_API_KEY`, `XAI_KEY`, `GROK_API_KEY` | `XAI_BASE_URL` | OpenAI-compatible |
| OpenRouter | `openrouter` | `OPENROUTER_API_KEY`, `OPENROUTER_KEY`, `OPENROUTER_TOKEN` | `OPENROUTER_BASE_URL` | OpenAI-compatible |
| Mistral | `mistral` | `MISTRAL_API_KEY`, `MISTRAL_KEY`, `MISTRAL_TOKEN` | `MISTRAL_BASE_URL` | OpenAI-compatible |
| Together | `together` | `TOGETHER_API_KEY`, `TOGETHER_KEY`, `TOGETHER_TOKEN` | `TOGETHER_BASE_URL` | OpenAI-compatible |
| DeepSeek | `deepseek` | `DEEPSEEK_API_KEY` | `DEEPSEEK_BASE_URL` | OpenAI-compatible |
| Fireworks | `fireworks` | `FIREWORKS_API_KEY` | `FIREWORKS_BASE_URL` | OpenAI-compatible |
| Perplexity | `perplexity` | `PERPLEXITY_API_KEY` | `PERPLEXITY_BASE_URL` | OpenAI-compatible |
| Moonshot | `moonshot`, `kimi` | `MOONSHOT_API_KEY`, `KIMI_API_KEY` | `MOONSHOT_BASE_URL` | OpenAI-compatible Moonshot/Kimi endpoint |

Notes:

- `--api-key` overrides the environment for the current invocation.
- `create` can also run with `deterministic` when you do not want a remote provider.
- Daemon-backed commands forward provider settings into host-backed runs; switching keys does not require a daemon restart.

## Environment

| Variable | Description |
|----------|-------------|
| `ANIMA_DAEMON_URL` | Override the daemon base URL for CLI commands. Defaults to `http://127.0.0.1:8080`. |
| Provider key env vars | Credentials used for generation or daemon-backed execution. See the provider table above. |
| Provider base URL env vars | Override provider endpoints such as `OPENAI_BASE_URL`, `OLLAMA_BASE_URL`, or `OPENROUTER_BASE_URL`. |
| `ANIMAOS_WORKSPACE_ROOT` | Workspace root used by mod configuration and workspace-aware tooling. |

## Good Defaults

- Use `create` when you want a durable human-editable workspace.
- Use `launch` when you want the best operator experience and the TUI.
- Use `run` when you only need a quick one-off answer.
- Use `chat` when you want an agent REPL without creating a workspace first.
- Use `agents` when you are debugging daemon-held state.
