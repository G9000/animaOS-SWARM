An **agent** in animaOS is an autonomous LLM-powered runtime with a configuration, state, and optional tools. Agents can run tasks individually or be composed into [swarms](/sdk/swarms/) for multi-agent coordination.

## In This Section

- [Lifecycle & Retry](/sdk/agents/lifecycle/) covers create, run, inspect, recent-memory reads, and safe retry keys.
- [Actions & Plugins](/sdk/agents/tools/) covers typed actions, plugin bundles, and registration patterns.

## Agent Configuration

The `AgentConfig` interface defines everything an agent needs to operate. Use the `agent()` helper for typed autocompletion.

```typescript
import { agent } from '@animaOS-SWARM/sdk';

const config = agent({
  name: 'researcher',
  model: 'gpt-4o-mini',
  provider: 'openai',
  system: 'You are a research assistant. Provide concise, factual answers.',
  bio: 'An AI research assistant',
  lore: 'Created to help users find information quickly.',
  knowledge: ['machine learning', 'neural networks'],
  topics: ['AI', 'science'],
  adjectives: ['curious', 'precise'],
  style: 'Concise and technical',
  tools: [],
  plugins: [],
  settings: {
    temperature: 0.7,
    maxTokens: 2048,
    maxRetries: 3,
  },
});
```

### `AgentConfig` fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | `string` | ✅ | Unique identifier for the agent |
| `model` | `string` | ✅ | LLM model identifier (e.g. `gpt-4o-mini`) |
| `provider` | `string` | — | Provider key: `openai`, `anthropic`, `ollama`, `openrouter` |
| `system` | `string` | — | System prompt that defines behavior |
| `bio` | `string` | — | Short biography used in character building |
| `lore` | `string` | — | Background story / origin context |
| `knowledge` | `string[]` | — | Domain knowledge topics |
| `topics` | `string[]` | — | Conversation topics the agent focuses on |
| `adjectives` | `string[]` | — | Personality descriptors |
| `style` | `string` | — | Response style guidance |
| `tools` | `Action[]` | — | Tools the agent can invoke (see [Actions](#actions)) |
| `plugins` | `Plugin[]` | — | Bundled actions, providers, and evaluators |
| `settings` | `AgentSettings` | — | Model hyperparameters (see below) |

### `AgentSettings`

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `temperature` | `number` | — | Sampling temperature (0–2) |
| `maxTokens` | `number` | — | Maximum tokens per response |
| `timeoutMs` | `number` | — | Request timeout in milliseconds |
| `maxRetries` | `number` | — | Retry count on transient failures |

<Aside type="note">
  `AgentSettings` also accepts arbitrary additional keys via `[key: string]: unknown`, allowing provider-specific parameters.
</Aside>

## Create an Agent

```typescript
const myAgent = await client.agents.create(config);

// Returns: AgentSnapshot
// {
//   state: AgentState,
//   messageCount: number,
//   eventCount: number,
//   lastTask: TaskResult<Content> | null
// }
```

The agent is persisted by the daemon and can be retrieved later by its `state.id`.

## List Agents

```typescript
const agents = await client.agents.list();

for (const snapshot of agents) {
  console.log(
    `${snapshot.state.id}: ${snapshot.state.name} (${snapshot.state.status})`
  );
}
```

### `AgentStatus`

| Value | Meaning |
|-------|---------|
| `idle` | Waiting for input |
| `running` | Currently processing a task |
| `completed` | Last task finished successfully |
| `failed` | Last task failed |
| `terminated` | Agent was stopped |

## Get Agent Details

```typescript
const snapshot = await client.agents.get('agent-uuid-here');

console.log('Name:', snapshot.state.name);
console.log('Status:', snapshot.state.status);
console.log('Messages:', snapshot.messageCount);
console.log('Events:', snapshot.eventCount);
console.log('Token usage:', snapshot.state.tokenUsage);
```

### `AgentState`

| Field | Type | Description |
|-------|------|-------------|
| `id` | `UUID` | Unique agent identifier |
| `name` | `string` | Agent name |
| `status` | `AgentStatus` | Current runtime status |
| `config` | `AgentConfig` | Full configuration snapshot |
| `createdAtMs` | `number` | Unix timestamp (ms) |
| `tokenUsage` | `TokenUsage` | Cumulative token consumption |

### `TokenUsage`

| Field | Type | Description |
|-------|------|-------------|
| `promptTokens` | `number` | Tokens sent to the model |
| `completionTokens` | `number` | Tokens generated by the model |
| `totalTokens` | `number` | Sum of prompt + completion |

## Run an Agent

The `run` method sends a `Content` message to the agent and returns its result.

```typescript
const run = await client.agents.run('agent-uuid-here', {
  text: 'Research the latest advances in quantum computing',
});

if (run.result.status === 'success') {
  console.log('Result:', (run.result.data as { text: string }).text);
} else {
  console.error('Run failed:', run.result.error);
}
```

### `Content` shape

```typescript
interface Content {
  text: string;
  attachments?: Attachment[];
  metadata?: Record<string, unknown>;
}

interface Attachment {
  type: 'file' | 'image' | 'url';
  name: string;
  data: string;
}
```

### Retrying a run safely

When the Rust daemon is configured with Postgres persistence, completed tool steps can be reused on a retried run. To opt into that behavior, send a stable retry key in `metadata` and repeat the same value when retrying the same logical request.

```typescript
const run = await client.agents.run('agent-uuid-here', {
  text: 'Research the latest advances in quantum computing',
  metadata: {
    retryKey: 'quantum-brief-2026-05-04',
  },
});
```

- Reuse only happens when you resend the same `retryKey` or `idempotencyKey`.
- Fresh runs should omit the key or use a new value.
- This is tool-step retry recovery, not full daemon process resume.

### `AgentRunResponse`

| Field | Type | Description |
|-------|------|-------------|
| `agent` | `AgentSnapshot` | Updated agent snapshot after the run |
| `result` | `TaskResult<Content>` | Task execution result |

## Actions

Actions are the primary extension point for agent capabilities. Each action is a typed tool the LLM can invoke.

```typescript
import { action } from '@animaOS-SWARM/sdk';

const memorySearch = action({
  name: 'memory_search',
  description: 'Search stored memories by keyword',
  parametersSchema: {
    query: { type: 'string', description: 'Search query' },
    limit: { type: 'number', description: 'Maximum matches to return' },
  },
  handler: async (_runtime, _message, args) => {
    const query = typeof args.query === 'string' ? args.query : '';
    return {
      status: 'success',
      data: { text: `Search requested for: ${query}` },
      durationMs: 0,
    };
  },
});
```

### `Action` interface

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | `string` | ✅ | Unique action identifier |
| `description` | `string` | ✅ | LLM-visible description |
| `parametersSchema` | `Record<string, unknown>` | ✅ | JSON Schema parameters |
| `handler` | `(runtime, message, args) => Promise<TaskResult>` | ✅ | Execution logic |
| `validate` | `(runtime, message) => Promise<boolean>` | — | Availability check |
| `examples` | `ActionExample[]` | — | Few-shot examples for the LLM |

### Registering actions on an agent

```typescript
const myAgent = await client.agents.create(
  agent({
    name: 'researcher',
    provider: 'openai',
    model: 'gpt-4o-mini',
    system: 'You are a research assistant.',
    tools: [memorySearch, webFetch, todoCreate],
  })
);
```

<Aside type="tip">
  For complex agents, bundle related actions into a [Plugin](/sdk/reference/#plugin) instead of listing them individually.
</Aside>

## Plugins

A `Plugin` groups actions, providers, and evaluators into a reusable module.

```typescript
import { plugin } from '@animaOS-SWARM/sdk';

const researchPlugin = plugin({
  name: 'research',
  description: 'Web search and memory tools for research agents',
  actions: [memorySearch, webFetch],
  providers: [currentTimeProvider],
  evaluators: [citationEvaluator],
  init: async (runtime) => {
    console.log('Plugin initialized for', runtime.agentId);
  },
});
```

### `Plugin` interface

| Field | Type | Description |
|-------|------|-------------|
| `name` | `string` | Plugin identifier |
| `description` | `string` | Human-readable description |
| `actions` | `Action[]` | Tools provided by the plugin |
| `providers` | `Provider[]` | Dynamic context providers |
| `evaluators` | `Evaluator[]` | Post-response evaluators |
| `init` | `(runtime) => Promise<void>` | Lifecycle hook on registration |
| `cleanup` | `(runtime) => Promise<void>` | Lifecycle hook on shutdown |

## Recent Memories

Retrieve the most recent memories for a specific agent:

```typescript
const memories = await client.agents.recentMemories('agent-uuid-here', {
  limit: 10,
});

for (const memory of memories) {
  console.log(`[${memory.importance}] ${memory.type}: ${memory.content}`);
}
```

### `AgentMemory` fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Memory identifier |
| `agentId` | `string` | Owning agent |
| `agentName` | `string` | Agent name at creation time |
| `type` | `string` | Memory type (e.g. `fact`, `reflection`) |
| `content` | `string` | Memory text |
| `importance` | `number` | Importance score (0–1) |
| `createdAt` | `number` | Unix timestamp (ms) |
| `tags` | `string[] \| null` | Optional tags |
| `scope` | `'shared' \| 'private' \| 'room'` | Visibility scope |

## Full Example: Research Agent with Tools

```typescript
import { agent, action, plugin, createDaemonClient } from '@animaOS-SWARM/sdk';

const client = createDaemonClient();

const webFetch = action({
  name: 'web_fetch',
  description: 'Fetch and summarize a URL',
  parametersSchema: {
    url: { type: 'string', description: 'URL to fetch' },
  },
  handler: async (_runtime, _message, args) => {
    const url = String(args.url);
    // ... fetch and summarize logic
    return {
      status: 'success',
      data: { text: `Summary of ${url}` },
      durationMs: 1200,
    };
  },
});

const researchPlugin = plugin({
  name: 'research',
  description: 'Web research tools',
  actions: [webFetch],
});

const researcher = await client.agents.create(
  agent({
    name: 'deep-researcher',
    provider: 'openai',
    model: 'gpt-4o',
    system: 'You are a thorough research assistant. Use tools to verify facts.',
    plugins: [researchPlugin],
    settings: { temperature: 0.3, maxTokens: 4096 },
  })
);

const run = await client.agents.run(researcher.state.id, {
  text: 'What are the latest breakthroughs in fusion energy?',
});

console.log(run.result.data?.text);
```
