The CLI accepts the same main provider family across `create`, `launch`, `run`, and `chat`. `create` also supports `deterministic` for local generation without a remote provider.

## Runtime Environment

| Variable | Description |
|----------|-------------|
| `ANIMA_DAEMON_URL` | Override the daemon base URL for daemon-backed CLI commands |
| Provider key env vars | Credentials used for generation or execution |
| Provider base URL env vars | Override provider endpoints such as `OPENAI_BASE_URL` or `OLLAMA_BASE_URL` |
| `ANIMAOS_WORKSPACE_ROOT` | Workspace root used by mod configuration and workspace-aware tooling |

## Override Rules

- `--api-key` overrides the environment for the current CLI invocation.
- `create` can use `deterministic` when you want local generation without a remote model.
- Daemon-backed commands forward provider settings into the host-backed run; changing keys does not require a daemon restart.

## Common Examples

```bash
# Local Ollama
OLLAMA_BASE_URL=http://127.0.0.1:11434/v1 \
animaos run "Summarize the plan" -p ollama -m qwen2.5:7b

# Staging daemon
ANIMA_DAEMON_URL=https://staging.example.com \
OPENAI_API_KEY=sk-... \
animaos agents list

# Deterministic local workspace generation
animaos create agency local-team -p deterministic --yes
```

<Aside type="tip">
  The clean default is simple: keep provider credentials in the environment, use `--api-key` only for temporary overrides, and use `ANIMA_DAEMON_URL` only when you intentionally want a different host target.
</Aside>
